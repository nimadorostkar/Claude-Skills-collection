# Examples

Worked sessions showing skills in use, end to end. Each shows which skills loaded, what they changed about the agent's approach, and the outcome.

| Example | Skills | Outcome |
| --- | --- | --- |
| [Debugging a slow endpoint](slow-endpoint.md) | `database-performance`, `postgres` | 3,184ms → 1.8ms |
| [Reviewing a pull request](pull-request-review.md) | `code-review`, `secure-coding` | One blocking finding no scanner would catch |
| [Fixing a RAG system](rag-diagnosis.md) | `rag`, `llm-evaluation` | The prompt was never the problem |

The point of these is not the outcome. It is the *order* — measure before optimizing, diagnose before fixing, evaluate the component before the whole. That order is what the skills enforce, and it is what an agent skips without them.
